How about FLUX?

Store

  • Holds the data of an application
  • Data in a store must only be mutated by responding to an action
  • Every time a store's data changes it must emit a "change" event

                {
                  users: [{id: 1, name: "Bob"},{id: 2, name: "Kate"}],
                  selectedUser: {id: 1, name: "Bob"}
                }

View

  • Subscribe to events emitted by store (re-render)
  • Dispatches an action when user interacts with the UI

Action

  • Intention to change application state
  • define the internal API of your application

                              {
                                type: "ADD_USER",
                                payload: {id: 3, name: "John"}
                              }

Dispatcher

  • Acts when new action is dispatched
  • Produce new state of application

                reduce(state, action) {
                  switch (action.type) {
                    case "ADD_USER":
                      return [...state.users, action.payload]
                    default:
                      return state;
                  }
                }
